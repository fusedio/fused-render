# fused-render — Decision Log & Project Context

Purpose: this file + `SPEC.md` + `ARCHITECTURE.md` carry the *entire* design conversation.
A fresh session should be able to continue the project from these three files alone.

---

## What we are building (one paragraph)

A fully local file explorer: a Python server on `127.0.0.1` + a browser UI to browse the **entire computer's** filesystem and preview files. The core primitive is **renderable HTML**: any `.html` file previews live and gets an injected JS runtime with (a) `fused.runPython(pyPath, params)` — executes the `main()` function of a local Python file and returns its JSON result — and (b) `fused.params` — string key/values synced into the browser URL, so any view's state is bookmarkable/refreshable. **Preview templates** (parquet table, image, text …) are just renderable HTML files shipped inside the app that receive the target file as a reserved `_file` param — the same two primitives are the entire rendering power of the system. Local only, forever; never cloud — reframed, not overturned, by mounts (D102): the app itself still only ever reads local absolute paths; remote storage enters exclusively as an rclone-managed *mount* presenting as a local folder, so nothing downstream of the mount layer speaks a cloud protocol.

### Editing decisions (2026-07-05)

| # | Decision | Choice | Rationale / rejected alternatives |
|---|---|---|---|
| D36 | Cross-origin POST guard | The two mutating/executing endpoints (`POST /api/run`, `POST /api/fs/write`) require a custom `X-Fused: 1` request header; missing/wrong → 403 | Read endpoints are already safe cross-origin (the browser blocks a foreign page from reading our response), but a POST can be fired blind via a no-cors fetch by any website open in the same browser. Cheap and removable if the descoped security layer lands. |
| D37 | File editing | `POST /api/fs/write` (atomic temp-file + `os.replace`, preserves mode) with optimistic locking on `st_mtime`; `code_template.html` is now an always-editable CodeMirror buffer (Save / Cmd+S, conflict banner offering Reload/Overwrite) | Supersedes SPEC §1 non-goal "no editing in v1". | D38 | Bookmark updating | Opening a bookmark **via sidebar click** arms it (sessionStorage `fused.armedBookmark`); when view params diverge from the saved url on the same page, an "Update bookmark" button appears left of "+ Bookmark" and overwrites the bookmark's url with the current one. ### Layout URL grammar — parenthesized `_layout` (2026-07-06, spec = SPEC §14.1 LM-2)

| # | Decision | Choice | Rationale / rejected alternatives |
|---|---|---|---|
| D51 | `_layout` scope grammar | The entire `_layout` value is **parenthesized and emitted last**: `?global=1&_layout=(/a.html?x=1&y=2,/b.html?z=3)`. Parens delimit param scope visually (inside = iframe-local, outside = global) and structurally: **`&` is literal inside them** — segment queries read exactly as written, no `%26` soup. Wrap is codec-transparent (`parseLayout` already reads `(A,B)` as `A,B`); only the URL layer changed. All shell-query reads go through `layout-codec.js` `splitShellSearch` (balanced-paren scan; safe — literal parens inside segments are codec-escaped, so span parens are structural and balanced): panel.js, tabs.js, breadcrumb's bookmark-equality `sameSearch`. The runtime (standalone, no imports) duplicates the scan as `splitSearch`: get/getAll parse only the non-layout remainder; `set()` reinserts the raw span untouched and last — also fixes the old wart where `URLSearchParams.toString()` percent-mangled a readable `_layout` on the first `fused.params.set()`. **Strict read** (owner call): unwrapped `_layout` = not this grammar, reads as absent, key dropped on next sync — old-format bookmarks fall back to a single start-dir pane/tab. **Paste breakage accepted** (owner call): auto-linkers may eat the trailing `)`; unbalanced span = invalid layout (span still excluded from params), missing-layout fallback | Chosen over "enforce `_layout` last as position-only convention" (look without guarantee; `&` stays escaped) and over "raw tail after `_layout=`" (position-dependent; trailing hand-added `&debug=1` silently swallowed into last segment — parens make position a convention and the boundary explicit, params after `)` stay global). Lenient read of old grammar rejected (owner call): one grammar, no two-format parse paths. Auto-closing missing trailing parens rejected with it: same leniency, masks truncation |

### Tile-daemon CORS + access token (2026-07-20)

| # | Decision | Choice | Rationale / rejected alternatives |
|---|---|---|---|
| D122 | **Open CORS (`Access-Control-Allow-Origin: *`) on the standalone tile daemons, gated by a per-daemon access token — distinct posture from D36's main-server guard** | The four persistent localhost tile daemons — `geotiff/tile_server.py`, `netcdf/grid_tile_server.py`, `map/vector_tile_server.py`, `zarr_aoi/tile_server.py` — each bind their own random port on `127.0.0.1` (so `runPython`'s ~700ms per-call subprocess cost doesn't cap MapLibre pan/zoom responsiveness) and answer every endpoint with `Access-Control-Allow-Origin: *`. Endpoints are read-only GETs (`/meta`, `/tile/{z}/{x}/{y}.png` or `.mvt`, `/hist`, `/value`/`/probe`, `/stats`; `map/`'s `/open` is the one POST, an async warm-up); templates run in the shell's iframe (a different port = different origin), so their `<img>`/`fetch`/MapLibre requests need the response readable cross-origin, which is why the D36 no-CORS trick can't be reused here. **Access control is a per-daemon token, not CORS.** Each daemon mints `secrets.token_urlsafe(32)` at startup, writes it into its state file (`~/.cache/fused-render-*/daemon.json`) next to the port, and returns it from `main(action="ensure")` alongside the port. Every endpoint except `/ping` (a version-only liveness probe) requires `?t=<token>`; a missing/wrong token is a 403 before any dispatch. The templates thread the token into every request: the three sci-viz templates append it in their one `dURL()` chokepoint, `pyramid/` appends it to its `/ltile` URL, and `map/`'s `map_render.py` folds it into the `tile_url`/`status_url`/`meta_url` query strings it hands the client. Token must ride the query string (not a header) because `<img>`/MapLibre tile sources can't set custom headers. `/quit` is token-gated too (the respawn path in `main()` passes the stale daemon's own token); `/ping` stays open so liveness needs no secret. | Contrast with D36 (main `server.py`): D36 withholds CORS headers to make a custom-header preflight fail, blocking blind cross-origin POSTs — that works only because the main server's same-origin shell JS never needs to read a foreign response. The tile daemons have the opposite requirement (their only client *is* a cross-port iframe reading tile bytes), so open CORS is deliberate, not an oversight. But `127.0.0.1` binding is **not** an access boundary against the real threat here — a malicious page in the *same browser* can fetch `http://127.0.0.1:<port>/…` cross-origin, and open CORS lets it read the reply; loopback only stops a different *machine*. Originally (first cut of this decision) the only obstacle was the random ephemeral port — obscurity, not access control. The token closes that: a foreign page can't produce a valid request even if it guesses the port, so the read-only tile/metadata bytes stop being cross-origin-readable. The token lives in the state file, so it's only as private as the local filesystem — consistent with D3 (local code execution is already out of scope; this guards the *browser* boundary, which the port-only scheme left open). Rejected a header-based token (breaks `<img>`/MapLibre tile loads); rejected reflecting the main origin instead of `*` (needs the daemon to know the main port, and the token already gates access so CORS breadth no longer matters). |
