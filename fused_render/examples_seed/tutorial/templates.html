Tutorial — custom templates

Every preview you've seen in Fused — the CSV table, the map, the Claude chat — is a template: a folder with an HTML page and optional Python, exactly the pattern your own pages use. Which means you can replace any of them, or add your own.

1The management UI, live

Templates are managed at /view/_templates (also in the sidebar footer). Here it is, embedded — two tabs: File bindings maps extensions to templates, Library holds the templates themselves.

Templates management — bindings on one tab, the library on the other open in the app ↗

2Anatomy of a template

A template is a folder. One file is required; everything else is optional.

the folder
my_template/
├─ template.html   # required — the view
├─ icon.svg        # mode-switcher icon
├─ condition.py    # gate: offer only when true
└─ helpers.py      # any python the view calls
where folders live
~/.fused-render/templates/          # yours
~/.fused-render/.core-templates/    # built-ins (managed,
                                    # refreshed on upgrade)
The page gets the file it's previewing as a _file param and uses the same fused global as your own pages — fused.runPython("./helpers.py", …) and all.

3Override a built-in

Don't like the stock CSV view? Shadow it. A user template with the same name as a core one wins, everywhere that name is bound.

1

Find it in the Library

The Library tab groups core and user templates. Export the core one you want to change as a zip.

2

Recreate it under your dir

Import the zip (or use New template) so a template with the same name lands in ~/.fused-render/templates/.

3

Edit — it just wins

Name-shadowing does the rest: every file type bound to that name now renders with your version. Delete your copy to fall back to core.

4Bind an extension

The File bindings tab is a table of key → ordered template list; the first entry is the default view, the rest appear in the mode switcher. Click a row to edit — chips drag to reorder — or add a new key. Keys come in four shapes:

.csvplain extension — the common case
.geo.parquetcompound extension — more specific, wins over .parquet
.*.jsononly compound json files (a.b.json), not every .json
.zarr/trailing slash — matches directories named like the key

Your binding for a key fully replaces the core list for that key — it doesn't merge. The row editor's Reset to core undoes your override; Disable for type empties the list so the type falls back to the raw default.

5New template, built by Claude

The Library's New template button scaffolds a working starter — and ships authoring skills in its .claude/skills/. Click Open in Claude on the success screen and a Terminal opens with Claude Code sitting in the new folder, ready to build the view for you. That workflow is covered in the claude.html guide.

Sharing goes both ways: Export zips any of your templates; Import is a two-step wizard that can read a bundled recommendation.json and offer the author's suggested bindings as step two.

6Precedence, in one line

User beats core, first chip wins.
A user template shadows a core one of the same name; a user binding replaces the core list for that key; the first template in a binding is the default view.
