Tutorial — Claude Code in Fused

Every file preview has a Claude mode: a chat scoped to that one file, powered by the Claude Code CLI on your machine. Ask about the file, or ask for changes — Claude edits it in place, and the page reloads with the result. Part of the tutorial series ↗

1What Claude does here

Open any file, and the mode switcher in the preview header offers Claude alongside the other views. It isn't a generic chatbot: each conversation is pinned to the file you were looking at, Claude runs in that file's folder, and edits it makes land directly on disk.

Prerequisite: the claude CLI.

Fused runs Claude Code installed on your own machine — nothing here talks to a Fused server. If claude isn't on your PATH, the chat below still opens, but sending a message will tell you to install it first.

2Try it, live

This is Claude mode opened on the sine/ example next door — the same view you'd reach from its mode switcher. Ask it something about the file ("what does the slider control?") right here.

sine/sine.html — in Claude mode (?_mode=claude) open in the app ↗
The sine/ example isn't next to this tutorial. Open any HTML or parquet file from your workspace and pick Claude from the mode switcher instead.

3Your first session

Three clicks from any file to a working conversation.

1

Open a file, switch mode

Click a file in the explorer, then pick the Claude icon from the mode switcher in the preview header.

2

Ask, or ask for changes

Claude answers about the file — and can edit it. It runs with acceptEdits, so requested changes are written to disk immediately, no approval prompt.

3

Come back anytime

Sessions are saved per file (in its .json sidecar) and survive the file being moved. The Sessions list on the chat home resumes any of them.

4Open in Claude — for templates

There's a second, bigger door. In Templates management, every template you own has an Open in Claude button: it opens a Terminal window with Claude Code already running inside that template's folder — a full agent session for building the whole template, not just one file. (macOS only; it drives Terminal.app.)

# what "Open in Claude" amounts to:
cd ~/.fused-render/templates/my_template
claude
New templates scaffold with authoring skills in .claude/skills/, so the Claude session that opens already knows how Fused templates work. More on this in the custom templates tutorial.

5Tips

Pick a model & effortTwo dropdowns on the chat home (and in the footer while chatting) select the model — default, fable, opus, sonnet, haiku — and the reasoning effort.
Resume from SessionsThe chat home lists this file's past sessions. Opening one restores the full transcript and continues where you left off.
Deep-link a sessionA session's URL carries _mode=claude&session_id=… — bookmark or share it and it reopens on exactly that conversation.
Arrives from annotationsThe annotate view's "Send to Claude" button opens this same chat pre-loaded with your review comments — see the annotations tutorial.
