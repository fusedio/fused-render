Start here

Fused Render turns any folder on your machine into a small app: an HTML page for the interface, Python files for the work. Four ideas cover the whole product — each one is live on this page.

1A page + Python, in a folder

This page is index.html. Next to it sits roll.py. One call connects them: the page asks, your local Python answers with JSON.

Python hasn't rolled yet.
# roll.py — the whole file
def main(count: int = 5):
    import random
    faces = [random.randint(1, 6)
             for _ in range(count)]
    return {"faces": faces,
            "total": sum(faces)}
The page calls it with await fused.runPython("./roll.py"). No server code, no decorator — a function named main() is the whole contract.

You don't write a page for every file. Built-in templates render each file type the moment you click it — and a template is itself just an HTML + Python project like this one.

Forty-plus templates ship built in. A file type can carry several modes — a CSV offers the table and the raw code view; a switcher in the preview header flips between them. And you can override any of it: drop a folder in ~/.fused-render/templates/, add one line to its registry.json, and your template wins for that extension.

3Python runs on your machine

Not a sandbox, not a cloud runtime — the Python is local, so it sees your real files. It can read data with pandas or DuckDB, shell out, write results. Proof: ask it what's around this very page.

      4When it's ready, publish it

      A project stays private on localhost until you say otherwise. Deploying is one explicit action from the preview header — nothing leaves your machine before you click.

      1

      Preview what ships

      The Deploy dialog lists exactly what would publish: the page, each .py it calls, each data file it reads. Add or remove files before anything happens.

      2

      Export a bundle

      Your folder is packed into a portable bundle — same layout as the folder itself, plus a manifest. runPython and params keep working; the page needs no edits.

      3

      Get a URL

      The fused CLI publishes the bundle to hosted serving at open.fused.io. Redeploy after edits, or revoke it — the URL and its status live in the same dialog.

      If a page can't deploy cleanly — say it calls runPython with a computed path the bundler can't trace — the dialog says so up front, before publishing, never as a broken URL after.
