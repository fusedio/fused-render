Tutorial — using Fused

Fused Render turns folders on your machine into live pages: any HTML file renders as an app, and a fused global lets it run the Python sitting next to it. This tutorial is itself one of those pages — everything below runs for real.

This is the sine/ example from your workspace, embedded live. Drag its slider — the Python next to it recomputes the curve right here, inside the tutorial.

2HTML calls Python

Next to this page sits hello.py. The button below sends your input to it and renders what comes back — that round trip is the whole product.

Python hasn't said anything yet.
# hello.py — the whole contract
def main(name: str = "world"):
    return {"greeting": f"Hello, {name}!"}
And the page's side is one line: await fused.runPython("./hello.py", {name}). No server, no route, no decorator — a function named main() is all it takes.

3State lives in the URL

Controls don't keep private state — they write to the page's URL through fused.params. Refresh, bookmark, or share the link and the page comes back exactly as you left it.

this page's URL now ends in  ?level=40
Move the slider, then refresh this page — the level survives, because the URL is the state.

5Make your own

A project is a folder with a page and a Python file. That's the entire setup.

1

Make a folder

Anywhere Fused Render can see — ~/Documents/Fused/my_tool/ is a fine home.

2

Drop in two files

An index.html for what you see, a tool.py for the work. Copy the pair below to start.

3

Click the HTML file

It renders as a live page immediately. Edit either file and the page reloads itself.

index.html
<!DOCTYPE html>
<html><body>
<h1 id="out">…</h1>
<script>
  fused.runPython("./tool.py", { n: "6" })
    .then(r => out.textContent = r.answer);
</script>
</body></html>
tool.py
def main(n: int = 3):
    return {"answer": f"{n} × {n} = {n * n}"}
Type-annotated inputs arrive as real numbers; whatever main() returns comes back to the page as JSON.
